# 嵌入式项目 Bug 汇总与改进方向

> 从本项目（STM32平衡车）的开发过程中，统计所有出现过的错误、修复方法，以及对应的代码行。

---

## 一、语法/语言层面

### 1. 在 C 语言中使用 C++ 风格的函数式强制转换
**问题**：`User/main.c` 第 69、72 行（旧版本）
```c
// ❌ 错误：int8_t(value) 是 C++ 语法，Keil C 编译器不认可
OLED_ShowNum(1,4,int8_t(count1*100)%100,2);
// error: #254: type name is not allowed
```
**修复**：改用 C 风格强制转换
```c
// ✅ 正确
OLED_ShowNum(1,4,(int8_t)(count1*100)%100,2);
// 或更安全的方式
OLED_ShowNum(1,4,(int)(count1*100)%100,2);
```

### 2. 重复 #include
**问题**：`User/main.c` 第 4 行和第 8 行
```c
#include "MPU6050.h"   // 第 4 行
#include "MPU6050.h"   // 第 8 行 ← 重复
```
**修复**：删除重复行。

## 二、类型/参数不匹配

### 3. float 隐式转换为 int32_t 导致小数被截断
**问题**：`User/main.c` 第 50 行
```c
float Angle=0;   // 定义
OLED_ShowSignedNum(1,1,Angle_PID.Actual,1);   // Actual 是 float
// 函数签名: void OLED_ShowSignedNum(uint8_t, uint8_t, int32_t, uint8_t)
// float → int32_t 隐式转换，直接截断小数部分，0.99 → 0
```
**修复**：显式转换，让意图明确
```c
OLED_ShowSignedNum(1,1,(int32_t)Angle_PID.Actual,1);
```

### 4. 负数表达式的结果传给无符号参数（导致乱码）
**问题**：`User/main.c` 第 52 行
```c
// Angle_PID.Actual = -5.23 时：
// (int)(-5.23*100)%100 = -523%100 = -23
// OLED_ShowNum 第三个参数是 uint32_t
// -23 转为 uint32_t → 4294967273（巨大正数）→ 显示屏乱码
OLED_ShowNum(1,4,(int)(Angle_PID.Actual*100)%100,2);
```
**修复**：取绝对值
```c
int dec = (int)(Angle_PID.Actual * 100) % 100;
if (dec < 0) dec = -dec;
OLED_ShowNum(1, 4, dec, 2);
// 或
OLED_ShowNum(1, 4, abs((int)(Angle_PID.Actual*100)%100), 2);
```

### 5. `OLED_ShowSignedNum` 显示长度计算错误
**问题**：`User/main.c` 所有调用处
```c
// OLED_ShowSignedNum 的实现：每个数字前固定显示 '+' 或 '-'
// Length=1 时，实际占用 2 列（符号列 + 1位数字）
// 但用户按"只占1列"布局显示，导致 '-' 号显示在上一列覆盖其他内容
OLED_ShowSignedNum(1,1,Angle_PID.Actual,1);  // 实际占用列1(符号)、列2(数字)
OLED_ShowString(1,3,".");                      // 列3 是 "."，正常
// 如果值为 -5，显示为 "-5.??" 五列宽度没问题
// 但如果值为 5，显示为 "+5.??" 也占五列宽度
```
**认知**：`OLED_ShowSignedNum(Line, Col, Num, Len)` 实际占用 `Len+1` 列。布局时要多留一列给符号位。

## 三、复制粘贴错误

### 6. 显示 count2 小数时误用了 count1
**问题**：`User/main.c` 第 255 行（旧版本）
```c
// 第 253 行：显示 count2 的整数
OLED_ShowSignedNum(1,7,count2,1);
// 第 254 行：小数点
OLED_ShowString(1,9,".");
// 第 255 行：❌ 应该显示 count2 的小数，却写了 count1
OLED_ShowNum(1,10,(int)(count1*100)%100,2);
```
**修复**：
```c
OLED_ShowNum(1,10,(int)(count2*100)%100,2);
```

### 7. BlueSerial.c：串口时钟和外设不匹配
**问题**：`Hardware/BlueSerial.c` 第 20、44、54、61 行
```c
// 第 20 行：开启 USART3 时钟
RCC_APB1PeriphClockCmd(RCC_APB1Periph_USART3, ENABLE);
// 第 44 行：❌ 初始化的却是 USART2
USART_Init(USART2, &USART_InitStructure);
// 第 54 行：中断向量也是 USART2
NVIC_InitStructure.NVIC_IRQChannel = USART2_IRQn;
// 第 61 行：使能 USART2
USART_Cmd(USART2, ENABLE);
// 第 44 行 GPIO：PB10/PB11 → 这是 USART3 的引脚
// USART2 的引脚是 PA2/PA3
```
**后果**：USART3 的时钟开了，但寄存器初始化的是 USART2；引脚配的又是 USART3 的引脚。整个模块无法工作。

**修复方向**：全文件统一。如果想用 USART2，改时钟为 `RCC_APB1Periph_USART2`、引脚改为 `PA2/PA3`。如果想用 USART3，把所有 `USART2` 改为 `USART3`，同时注意与 `usart.c`（已占用 USART3）冲突。

### 8. MyUSART 和 BlueSerial 同时占用 USART3 + PB10/PB11
**问题**：`Hardware/usart.c` 和 `Hardware/BlueSerial.c`
- `usart.c` 使用 USART3 + PB10/PB11
- `BlueSerial.c` 也使能了 USART3 时钟 + PB10/PB11
- 两个模块对同一个外设做了两次初始化，后者覆盖前者

**修复**：蓝牙模块改用 USART2（PA2/PA3），或 USART1（PA9/PA10），与调试串口分开。

## 四、中断/并发相关

### 9. 中断和主循环共享的变量未加 volatile
**问题**：`User/main.c` 第 21-24 行及所有版本
```c
float Angleacc=0, AngleGyro=0, Angle=0;
uint8_t RunFlag=0;
int8_t RIGHT_PWM, LEFT_PWM, AVE_PWM, DIF_PWM;
// 这些变量在 TIM3_IRQHandler 中写入，在 main() 的 while(1) 中读取
// 编译器可能将 main 中的读取优化为寄存器缓存，导致看不到 ISR 的更新
```
**修复**：
```c
volatile float Angleacc=0, AngleGyro=0, Angle=0;
volatile uint8_t RunFlag=0;
volatile int8_t RIGHT_PWM, LEFT_PWM, AVE_PWM, DIF_PWM;
```

### 10. 中断里做太重的浮点运算
**问题**：`User/main.c` TIM3_IRQHandler 第 57-96 行
```c
// 中断每 1ms 触发一次（假设 TIM3 周期 1ms），每 10 次做：
// MPU6050_GetData() — I2C 通信，约 400μs
// atan2() — 软件浮点，极慢（Cortex-M3 无 FPU）
// PID_Update() — 浮点乘法
// 全部在中断上下文中执行，阻塞其他中断
```
**修复方向**：中断里只设标志位、读传感器原始数据，浮点运算放到主循环。

## 五、算法逻辑

### 11. PID 积分项未累加（积分项永远为 0）
**问题**：`User/PID.c` 第 4-12 行
```c
void PID_Update(PID_t *p){
    p->Error1=p->Error0;
    p->Error0=p->Target-p->Actual;
    p->Out=p->Kp*p->Error0+p->Ki*p->ErrorInt+...;
    // ❌ 缺少: p->ErrorInt += p->Error0;
    // ErrorInt 始终为 0，Ki 设为任何值都无效
}
```
**修复**：
```c
void PID_Update(PID_t *p){
    p->Error1=p->Error0;
    p->Error0=p->Target-p->Actual;
    p->ErrorInt += p->Error0;           // ← 必须加上这一行
    p->Out=p->Kp*p->Error0+p->Ki*p->ErrorInt+p->Kd*(p->Error0-p->Error1);
    // 输出限幅...
}
```

### 12. PID 输出限幅无积分抗饱和
**问题**：`User/PID.c` 第 8-13 行
```c
if(p->Out > p->OutMax) { p->Out = p->OutMax; }
if(p->Out < p->OutMin) { p->Out = p->OutMin; }
// 输出被限幅了，但 ErrorInt 继续累加 → 积分饱和
// 现象：输出从 100% 恢复到正常需要很长时间（积分"退饱和"）
```
**修复**：输出被限幅时，同时回退积分项
```c
if(p->Out > p->OutMax) {
    p->Out = p->OutMax;
    p->ErrorInt -= p->Error0;   // 抗饱和退积分
}
if(p->Out < p->OutMin) {
    p->Out = p->OutMin;
    p->ErrorInt -= p->Error0;
}
```

### 13. 姿态角互补滤波公式存疑
**问题**：`User/main.c` 第 66 行
```c
AngleGyro = Angle - Gx/32768.0*2000*0.01;
// 标准写法应为：
// AngleGyro = Angle + gyro_rate * dt  （前向积分）
// 这里用的是 Angle - gyro_rate * dt，减去陀螺仪增量
// 符号方向取决于传感器安装方向，但一般为 "+"
```

### 14. 陀螺仪零偏硬编码
**问题**：`User/main.c` 第 64 行
```c
Gx += 125;   // 每次上电固定加125，不同传感器/温度下零偏不同
```
**修复方向**：开机时采样 500~1000 次取平均作为零偏值。

### 15. DIF_PWM（转向差速）始终为 0
**问题**：`User/main.c` 第 37 行
```c
int8_t RIGHT_PWM, LEFT_PWM, AVE_PWM, DIF_PWM;
// DIF_PWM 声明后从未被赋值，始终为 0
// LEFT_PWM  = AVE_PWM + DIF_PWM/2 = AVE_PWM
// RIGHT_PWM = AVE_PWM - DIF_PWM/2 = AVE_PWM
// 两个轮子永远同速，无转向控制
```

## 六、代码工程管理

### 16. 大量注释掉的旧代码未清理
**问题**：`User/main.c` 第 98-318 行
- 主文件 319 行，其中 221 行是注释掉的旧代码（70%）
- 包含多个版本的 main()、多个版本的 TIM3_IRQHandler()
- 严重影响可读性

**修复方向**：删掉旧代码，依靠 git 回退历史。

### 17. 函数命名不一致
**问题**：
- `motor.c` 最初为 `motor_SetSpeed`，后改为 `motor_SetPWM`（但旧注释代码还在引用旧名）
- `Encoder.c` 最初为 `Encoder_GetCounter1/2`，后改为 `Encoder_GetCounterL/R`

### 18. 项目配置 C99 模式最初未开启
**问题**：`project.uvprojx` `<uC99>0</uC99>`
- 未开启 C99 时，变量必须在函数开头声明，不能在代码中间声明
- **修复**：已改为 `<uC99>1</uC99>`

## 八、改进方向总结

### 🔴 最容易出错的三个地方：

1. **C/C++ 语法混用**
   - C 语言没有函数式强转 `type(value)`，只有 `(type)value`
   - 每次写完有类型转换的表达式，先确认是 `(type)expr` 格式

2. **有符号/无符号混用**
   - `int` 负数 → `uint32_t` 变成巨大正数（本项目的头号运行时 Bug）
   - 调用函数前检查参数类型：`OLED_ShowNum` 要 `uint32_t`，负数必须处理

3. **复制粘贴后忘记改关键变量**
   - 写完一段代码，检查每个引用的变量名是否正确
   - 写 USART 驱动时，对照检查：时钟、引脚、外设基地址、中断向量 4 个地方是否一致

### 🟡 中长期需要改进的习惯：

- **中断只做采集和标志位**，计算放到主循环
- **volatile 不要漏**：主循环读、中断写的变量一律加
- **上板调试前加串口打印**：用 `printf` 确认中间值，而不是盯着 OLED 猜
- **旧代码及时删**：git 已经有历史记录，注释掉的代码放里面只会混淆
- **设计结构体统一用 `{}` 初始化**：否则成员值是未知的
- **PID 写完后用阶跃响应测试**：验证 P/I/D 各自是否生效
