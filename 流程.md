# 平衡车项目开发流程

## 硬件方案

| 模块 | 型号 | 接口 | 说明 |
|------|------|------|------|
| MCU | STM32F103C8T6 | — | Cortex-M3, 64KB Flash, 20KB RAM, 72MHz |
| 姿态传感器 | MPU6050 | I2C1 (PB6-SCL, PB7-SDA) | 三轴陀螺仪+三轴加速度计 |
| 电机驱动 | TB6612FNG | TIM PWM + GPIO | 双路H桥 |
| 直流电机 | 带编码器减速电机 | TIM 编码器模式 | 霍尔编码器，用于速度闭环 |
| OLED | 0.96寸 I2C | PB8(SCL), PB9(SDA) | 软件I2C（已实现） |
| 电池 | 7.4V~12V 锂电池 | — | 给电机驱动板供电 |
| 蓝牙模块 | HC-05/HC-06 | USART1 (PA9/PA10) | 手机APP遥控（后期加） |

## STM32 引脚分配

```
OLED:     PB8(SCL), PB9(SDA)   — 软件I2C（已有）
MPU6050:  PB6(SCL), PB7(SDA)   — 硬件I2C1
电机A:    PA0(PWM-TIM2_CH1), PA4(AIN1), PA5(AIN2)
电机B:    PA1(PWM-TIM2_CH2), PB0(BIN1), PB1(BIN2)
编码器A:  PA6(TIM3_CH1), PA7(TIM3_CH2)
编码器B:  PB4(TIM3_CH1), PB5(TIM3_CH2)   — 用TIM4编码器模式
蓝牙:     PA9(TX), PA10(RX)    — USART1
控制中断: TIM4 或 SysTick       — 5ms 定时中断
```

## 控制理论核心

平衡车本质是倒立摆系统，需要三级串级PID控制：

```
┌─────────────────────────────────────────────┐
│                  遥控/蓝牙指令                │
│                     ↓                         │
│  ┌──────────┐    ┌──────────┐               │
│  │ 转向PID  │←───│ 速度PID  │   ← 编码器反馈  │
│  │ (最慢)   │    │  (中速)   │              │
│  └────┬─────┘    └────┬─────┘              │
│       │               │                     │
│       └───────┬───────┘                     │
│               ↓                             │
│       ┌───────────┐                         │
│       │  直立PID   │   ← MPU6050角度反馈      │
│       │   (最快)    │                       │
│       └─────┬─────┘                         │
│             ↓                               │
│       ┌───────────┐                         │
│       │  PWM输出   │  → 电机                  │
│       └───────────┘                         │
└─────────────────────────────────────────────┘
```

### 三个控制环的时序

| 控制环 | 频率 | 周期 | 控制算法 | 反馈来源 |
|--------|------|------|----------|----------|
| 直立环 | 200Hz | 5ms | PD | MPU6050 角度+角速度 |
| 速度环 | 50Hz | 20ms | PI | 编码器轮速 |
| 转向环 | 25Hz | 40ms | PD | 编码器差速 / 遥控指令 |

> 核心公式：`PWM = 直立环输出 + 速度环输出 + 转向环输出`

## 开发阶段

### 第一阶段：MPU6050 姿态解算

**目标：** 准确读取小车倾角，在 OLED 上实时显示

**新建文件：** `Hardware/MPU6050.c` + `MPU6050.h`

**要做的事：**
1. 硬件 I2C1 初始化（PB6/PB7）
2. 初始化 MPU6050（唤醒、配置量程）
3. 读取加速度计和陀螺仪原始数据
4. 姿态解算：互补滤波融合得到 Pitch 倾角

```
// 互补滤波核心公式
Angle = 0.98 * (Angle + Gyro_Y * dt) + 0.02 * Accel_Angle;
```

5. 通过 OLED 显示角度，用手转动模块验证数据正确

> **验证标准：** OLED 显示角度，水平=0°，前倾=正值，后仰=负值，无抖动无延迟

---

### 第二阶段：电机驱动

**目标：** 能控制两个电机正反转和转速

**新建文件：** `Hardware/Motor.c` + `Motor.h`

**要做的事：**
1. TIM2 CH1/CH2 配置为 PWM 输出（控制转速）
2. GPIO 控制 TB6612 方向引脚（AIN1/AIN2/BIN1/BIN2）
3. 封装 `Motor_SetSpeed(int16_t left, int16_t right)` 函数
4. 正负值对应正反转，绝对值对应转速

> **验证标准：** 调用 Motor_SetSpeed 函数，两轮能正反转、调速

---

### 第三阶段：直立环（核心关卡）

**目标：** 小车能自主站立不倒

**新建文件：** `Hardware/PID.c` + `PID.h` + `Hardware/Control.c` + `Control.h`

**要做的事：**
1. 实现通用 PID 结构体（Kp, Ki, Kd, 积分限幅等）
2. TIM4 中断，每 **5ms** 触发一次
3. 在中断中执行直立控制循环：
   - 读取 MPU6050 角度和角速度
   - PD 计算 → 得到 PWM 值
   - 设置电机 PWM

```
直立PWM = Kp * Angle + Kd * AngularVelocity
```

4. **调参**（直立环只有两个参数）：

| 步骤 | 操作 | 现象 |
|------|------|------|
| 1 | 先令 Kd=0，慢慢增大 Kp | 车开始有"硬度"，再大会高频振荡 |
| 2 | 车能"挣扎"但振荡 → 慢慢增大 Kd | 振荡逐渐消失，车平稳站立 |
| 3 | 微调 Kp, Kd | 达到用手轻推不倒下 |

> 调参时用 OLED 实时显示角度和 PWM 值，效率高很多

> **验证标准：** 车通电后能自主站立，轻推不倒下，倾斜后能自动恢复

---

### 第四阶段：速度闭环

**目标：** 小车能静止站立，不漂移

**新建文件：** `Hardware/Encoder.c` + `Encoder.h`

**要做的事：**
1. 配置 TIM3/TIM4 编码器模式读取轮速
2. 每 20ms 计算一次速度（在 Control 中实现）
3. 速度环 PI 控制，叠加到直立环输出上

```
最终PWM = 直立PD + 速度PI
```

| 步骤 | 操作 |
|------|------|
| 1 | 车已能站立，先加小 Kp_Speed |
| 2 | 车速趋于静止但还漂 → 加小 Ki_Speed |
| 3 | Ki 太大车身会前后大幅摇摆 → 减小 |

> **验证标准：** 车能原地静止站立，无明显前后漂移

---

### 第五阶段：蓝牙遥控 + 转向

**目标：** 能用手机遥控前进后退转弯

**要做的事：**
1. USART1 配置，蓝牙透传
2. 解析遥控指令（前进/后退/左转/右转/速度）
3. 转向环：左右轮差速控制
4. 指令转换为速度环目标值

```
左轮PWM = 直立PD + 速度PI - 转向值
右轮PWM = 直立PD + 速度PI + 转向值
```

> **验证标准：** 手机发送指令，小车能按指令方向移动

---

## 关键调参总结

| 参数 | 作用 | 太大会怎样 | 太小会怎样 |
|------|------|------------|------------|
| 直立 Kp | 角度响应力度 | 高频振荡 | 车身软绵、站不住 |
| 直立 Kd | 抑制抖动 | 电机高频噪音、抖动 | 无法抑制振荡 |
| 速度 Kp | 轮速响应 | 前后摇摆 | 归位慢 |
| 速度 Ki | 消除稳态误差 | 大幅前后摇晃 | 缓慢漂移 |
| 转向 Kp | 差速响应 | 转弯过猛 | 转弯无力 |

## 调试技巧

1. **OLED 是你的眼睛：** 把角度、角速度、PWM值、PID 参数都显示在 OLED 上，比串口打印直观得多
2. **按键调参：** 留两个按键，一个选参数，一个加减值，不用每次重新编译
3. **先开环再闭环：** 每个模块先单独测试通过，再组合
4. **手扶着调：** 直立环调参时用手扶住车，防止摔坏硬件
5. **角度零点要准：** 小车物理平衡点和 MPU6050 的零点必须对准，否则永远站不稳
